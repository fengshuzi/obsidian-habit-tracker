# 掌控习惯 - 快速开始指南

## 🚀 5分钟上手

### 第1步：部署插件

```bash
cd obsidian-habit-tracker
npm run build
npm run deploy
```

### 第2步：启用插件

1. 打开 Obsidian
2. 进入设置 → 第三方插件
3. 找到"掌控习惯"并启用

### 第3步：开始打卡

在你的日记文件中（如 `journals/2024-01-10.md`）添加习惯标签：

```markdown
# 2024年1月10日

今天完成了阅读和运动 #reading #exercise

早上冥想了10分钟 #meditation
```

### 第4步：查看统计

1. 点击左侧边栏的 ✓ 图标
2. 或使用命令面板（Cmd/Ctrl + P）搜索"打开掌控习惯"
3. 查看你的打卡记录和统计数据

## 📊 界面说明

### 顶部区域
- **标题**：显示应用名称（可自定义）
- **刷新数据**：手动刷新打卡数据
- **配置习惯**：打开配置界面

### 时间筛选
- **本周**：显示本周一到周日的数据
- **上周**：显示上周一到周日的数据
- **本月**：显示本月1号到月底的数据（默认）
- **上月**：显示上月1号到月底的数据

### 统计卡片
- **总打卡次数**：在选定时间范围内的总打卡次数
- **追踪习惯数**：配置的习惯总数

### 习惯统计
显示每个习惯的详细信息：
- 习惯名称
- 打卡次数
- 连续打卡天数（带火焰图标 🔥）

### 习惯列表
每个习惯显示：
- 习惯名称
- 打卡次数和连续天数
- 最近7天打卡圆点
  - 绿色圆点 = 已打卡
  - 灰色圆点 = 未打卡
  - 鼠标悬停显示日期

## ⚙️ 配置说明

### 基础设置
1. 点击"配置习惯"按钮
2. 切换到"基础设置"标签页
3. 输入自定义应用名称（如"我的习惯"）
4. 实时预览效果
5. 点击"保存"

### 习惯管理
1. 点击"配置习惯"按钮
2. 切换到"习惯管理"标签页
3. 点击"+ 添加新习惯"
4. 输入关键词（英文，如 `coding`）
5. 输入中文名称（如 `编程`）
6. 点击"保存"

## 💡 使用技巧

### 技巧1：灵活打卡
标签可以出现在句子的任何位置：

```markdown
今天完成了 #reading 和 #exercise，感觉很棒！

#meditation 早上冥想了10分钟

完成了今天的 #study 任务
```

### 技巧2：查看连续天数
- 连续打卡的习惯会显示火焰图标 🔥 和天数
- 保持连续打卡，挑战更高天数
- 中断后会重新开始计算

### 技巧3：使用时间筛选
- 默认显示本月数据
- 点击"本周"查看本周进度
- 点击"上月"回顾上月完成情况

### 技巧4：7天圆点
- 快速了解最近7天的打卡情况
- 绿色圆点表示已打卡
- 灰色圆点表示未打卡
- 鼠标悬停查看具体日期

## 🎯 最佳实践

### 1. 选择合适的习惯
- 不要追踪太多习惯（建议3-6个）
- 选择对你最重要的习惯
- 确保习惯是可衡量的

### 2. 每天记录
- 养成每天在日记中记录的习惯
- 可以在一天结束时统一记录
- 也可以完成后立即记录

### 3. 定期回顾
- 每周查看一次本周数据
- 每月查看一次本月数据
- 分析哪些习惯坚持得好，哪些需要改进

### 4. 保持简单
- 使用简短的关键词（如 `reading` 而不是 `reading-books`）
- 不要过度追求完美
- 重要的是坚持，而不是完美

## 🔧 常见问题

### Q1：为什么看不到打卡记录？
A：确保：
1. 日记文件在 `journals` 文件夹中
2. 文件名是 `yyyy-mm-dd.md` 格式（如 `2024-01-10.md`）
3. 使用了正确的标签格式（如 `#reading`）
4. 标签关键词在配置中存在

### Q2：如何修改应用名称？
A：
1. 点击"配置习惯"按钮
2. 切换到"基础设置"标签页
3. 输入新的应用名称
4. 点击"保存"

### Q3：连续天数为什么是0？
A：连续天数只有在今天或昨天有打卡时才会显示。如果中断了，会重新开始计算。

### Q4：可以修改日记文件夹路径吗？
A：可以在 `config.json` 中修改 `journalsPath` 字段，但需要重新加载插件。

### Q5：数据更新不及时怎么办？
A：
1. 点击"刷新数据"按钮手动刷新
2. 插件有30秒缓存，会自动更新
3. 修改日记文件后会自动清除缓存
